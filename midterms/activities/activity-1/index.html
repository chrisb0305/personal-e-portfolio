Activity 1: Variables

Christopher Balana | BSIT 2-5


This activity emphasizes the technical concepts of variable behavior and memory management in Java. It shows how variables can reference objects, how assignments redirect references, and how nullification affects accessibility. By analyzing these operations, the exercise highlights the importance of understanding how Java handles object creation, reference sharing, and memory usage, which are fundamental for writing efficient and logical programs.


For me, the task was easy because I already understood the basic concepts of variables and references. I was able to follow the logic without much difficulty. Overall, it felt straightforward and helped strengthen my grasp of how Java handles memory.

Part B: Explanation

Statement What Happens? Memory Impact
Employee e1 = new Employee("Anna", 20000); Creates a new Employee object that points to e1 with name "Anna" and salary 20000. employeeCount increments to 1. e1 → references Anna object in heap. employeeCount updated to 1.
Employee e2 = new Employee("Brian", 25000); Creates a new Employee object that points to e2 with name "brian" and salary = 25000. employeeCount increments to 2. e2 → references brian object. employeeCount incremented to 2.
Employee e7 = e1; e7 points to the same object at e1. e9 → references same object as e1 (Anna). e1 → reference) Anna · Brian's object is eligible for garbage collection.
e9.salary = 22000; Updates Anna's salary to 22000. e9 → salary → 22000. Since e1 also points to Anna, e1.salary is 22000 too. e1 → reference) Anna · Brian's object is eligible for garbage collection.
e4 = e1; e4 point to Anna's object. e4 → reference) Anna · e1 still reference Anna, e9 and e2 still reference brian's object is eligible for garbage collection.
e1 = NULL; e1 no longer references Anna e1 → NULL, e9 and e2 still reference Anna · brian's object is eligible for garbage collection.
e9.incrementSalary(e2); Cause incrementSalary on Anna's object, passing e2. Anna's valory increates by 12000 which overall equals to 27000. e9.salary → 27000. e1.salary is also 27000. employeeCount updated to 0.
e4 = NULL; e2 no longer references Anna. e2 → null. Anna still referenced by e9.
e9.display(); Prints Anna's details. e9 accesses Anna's object in the heap, retrieves its current state (name, salary, employeecount) and prints them.
